# Contributing to Claude TypeScript Cookbooks

Thank you for your interest! This guide walks you through setting up the development environment and running the TypeScript notebooks.

## Development Setup

### Prerequisites

- **Node.js** v18 or higher — [nodejs.org](https://nodejs.org/)
- **npm** v9 or higher (bundled with Node.js)
- **Python** 3.8 or higher with **Jupyter** (needed to run the kernel)
- **Anaconda / Miniconda** (recommended) or a plain `pip` install of Jupyter

> To check what you have: `node --version`, `npm --version`, `jupyter --version`

---

### Step 1 — Clone & Install Project Dependencies

```bash
git clone <your-repo-url>
cd claude-ts-cookbooks

# Install @anthropic-ai/sdk and all dev dependencies
npm install
```

---

### Step 2 — Set Up Your API Key

```bash
cp .env.example .env
# Open .env and replace the placeholder with your real key:
#   ANTHROPIC_API_KEY=sk-ant-...
# Get your key at: https://console.anthropic.com/settings/keys
```

---

### Step 3 — Install the TypeScript Jupyter Kernel (tslab)

**tslab** lets Jupyter notebooks execute TypeScript via Node.js. This is a **one-time global install**.

```bash
# Install tslab globally
npm install -g tslab

# Register the TypeScript (and JavaScript) kernel with Jupyter
tslab install
```

Verify the kernel is registered:

```bash
jupyter kernelspec list
# Expected output includes:
#   tslab   C:\Users\<you>\AppData\Roaming\jupyter\kernels\tslab
#   jslab   C:\Users\<you>\AppData\Roaming\jupyter\kernels\jslab
```

At this point the kernel is usable from a terminal via `jupyter notebook` or `jupyter lab`.  
If you are using **VS Code**, continue to the next section for additional setup.

---

## VS Code Setup

This section covers steps specific to running TypeScript notebooks inside **VS Code** with the
Jupyter extension. If you are running notebooks from a terminal (`jupyter lab` / `jupyter notebook`),
you can skip this section.

### Selecting the TypeScript Kernel

1. Open any `.ipynb` file in `claude-ts-cookbooks/`
2. Click **Select Kernel** (top-right corner of the notebook editor)
3. Choose **TypeScript (tslab)**

> **Tip:** If TypeScript doesn't appear in the kernel list after installing tslab, run  
> `Ctrl+Shift+P` → `Developer: Reload Window` to force VS Code to re-scan kernels.

---

### Running Notebooks

Each notebook is self-contained. Execute cells top-to-bottom:
- `Ctrl+Enter` — run current cell
- `Shift+Enter` — run current cell and advance
- **Run All** button — execute all cells in sequence

> **Note:** tslab runs TypeScript in the Node.js environment. The `@anthropic-ai/sdk` package
> is resolved from `claude-ts-cookbooks/node_modules` (installed in Step 1), so no extra
> install is needed per notebook.

---

### ⚠️ Troubleshooting: `spawn EINVAL` on Windows

**Symptom:** Executing any cell immediately fails with:

```
Error: Failed to start the Kernel 'TypeScript'. spawn EINVAL
```

**Cause:** This is a Windows-specific issue with VS Code's Jupyter extension. The default
`kernel.json` generated by `tslab install` sets `argv` to `["tslab.cmd", ...]`, but VS Code
cannot `spawn` a `.cmd` file directly — unlike a regular terminal which handles `.cmd` natively.

**Fix:** Wrap the command with `cmd /c` so Windows cmd.exe acts as the launcher.

Option A — Edit manually: open `%APPDATA%\jupyter\kernels\tslab\kernel.json` and set:

```json
{
  "argv": ["cmd", "/c", "tslab.cmd", "kernel", "--config-path", "{connection_file}"],
  "display_name": "TypeScript",
  "language": "typescript"
}
```

Option B — PowerShell one-liner:

```powershell
# ⚠️ Must use UTF-8 without BOM — PowerShell's default UTF8 adds a BOM that breaks Jupyter's JSON parser
$json = '{"argv":["cmd","/c","tslab.cmd","kernel","--config-path","{connection_file}"],"display_name":"TypeScript","language":"typescript"}'
[System.IO.File]::WriteAllText("$env:APPDATA\jupyter\kernels\tslab\kernel.json", $json, [System.Text.UTF8Encoding]::new($false))
```

After applying the fix: `Ctrl+Shift+P` → `Developer: Reload Window`, then re-select the kernel.

## Notebook Commit Policy

When you execute notebook cells, outputs and metadata are modified. **nbstripout** is configured
as a git filter to automatically strip both before every commit, keeping the notebook clean and
free of execution noise.

### nbstripout (auto-configured)

This repo uses **nbstripout** as a git filter. It is already set up via `.gitattributes`.
No manual steps needed — just `git add` and `git commit` as normal.

If you clone this repo on a new machine, run once:

```bash
pip install nbstripout
nbstripout --install
```

## Key Rules

1. **API Keys** — Never commit `.env`. Always use `.env.example` as the template.
2. **Models** — Use non-dated aliases: `claude-opus-4-6`, `claude-sonnet-4-6`, `claude-haiku-4-5`.
3. **Notebooks** — One concept per notebook. Keep outputs for demonstration purposes.
4. **TypeScript** — Strict mode is on. Type all function parameters and return values.
